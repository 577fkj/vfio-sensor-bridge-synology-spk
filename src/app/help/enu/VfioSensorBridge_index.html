VFIO Sensor Bridge Agent

Bridges hardware sensor data (temperature, fan speed, voltage, power) from PCI pass-through virtual machines back to the Proxmox VE host via virtio-serial. Sensors become visible to sensors, fancontrol, and the PVE monitoring dashboard.

Requirements

  • SimplePermissionManager — must be installed and activated.
  • The Synology NAS must run as a KVM/QEMU virtual machine on a Proxmox VE host.
  • The PVE host must have vfio-sensor-bridge-hostd and the vsb_hwmon.ko kernel module installed.
  • The QEMU VM configuration must include a virtio-serial channel named org.vfio_sensor_bridge.0.

Configuration Tabs

Status

Shows whether the package service and the agent process are running, whether SimplePermissionManager is active, and whether the virtio device is present. Use the Start/Stop/Restart buttons to control the agent process from this page.

Agent

Basic agent settings:

  • Virtio Port — path to the virtio-serial device.
  • Hwmon Scan Root — base directory for hwmon enumeration.
  • Rescan Interval — how often (seconds) to re-scan for new sensors.
  • Sample Interval — how often (seconds) to send sensor readings.
  • Heartbeat Interval — how often (seconds) to send a heartbeat to the host.

Sources

Configure additional sensor sources:

  • LSI HBA — reads temperatures from LSI/Broadcom SAS HBA controllers via /dev/mpt2ctl or /dev/mpt3ctl.
  • SmartCTL — reads disk temperatures using smartctl.

Persistent Sensors

Define sensors that continue to report a default value even when the underlying hardware source is temporarily unavailable (e.g. GPU suspended). Each persistent sensor maps to a specific hwmon chip and input.

Logs

Shows the last 80 lines of the agent log for troubleshooting, including startup errors, missing virtio device warnings, and agent exit codes.

Sensor Units

  • Temperature: milli-°C (65000 = 65 °C)
  • Fan speed: RPM
  • Voltage: milli-volts
  • Current: milli-amperes
  • Power: micro-watts
